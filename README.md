# バンド練習レビューサービス (MVP)

設計書 v0.2 に基づくPhase 1+2の実装(ローカル管理機能+共有機能)です。

## 含まれる機能

- 録音ファイルのアップロード → 無音検出による自動分割 → タイトル編集 → 保存
- Session一覧・Session詳細・Track再生
- ★お気に入り、タイムラインコメント(クリックでその位置にジャンプ)
- 共有リンクの発行(閲覧用 / 編集用)、共有相手はログイン不要で閲覧・コメント・お気に入り編集が可能

## 音声解析について

無音検出・分割ロジックは、既存の`rehearsal-rec-splitter.html`(build: 2026-07-10)の実コードから移植しています。

- RMS計算は0.2秒刻み、閾値は**絶対dBFS基準**(デフォルト-30dB。グローバル最大振幅基準ではありません)
- 「無音判定時間」「無音判定閾値」「最小演奏時間」の3スライダーで調整でき、「この設定で再判定」ボタンで(音声の再デコードなしに)高速に再判定できます
- WAV書き出しロジックも既存コードと同じです

## 含まれていない機能(今後の課題)

- **手動での分割点調整・可変速プレビュー・ズーム&ページング**: 既存ツールにあったこれらのインタラクティブな編集機能(波形タップでの分割点追加/削除、プレビュー再生の速度切替、5分/2分/1分ズーム)は、今回は移植していません。自動判定結果をそのまま使う前提のシンプルなレビュー画面になっています。必要であれば追加できます。
- **共有後のローカル編集の自動反映**: 一度共有した後、あなたの端末でタイトルやお気に入りを変更しても、共有側(R2)には自動で反映されません。Session詳細の「共有リンクを表示」→「更新をアップロード」で手動反映してください(共有相手が追加した内容は上書きされる場合があります)。
- **複数Band対応**: 1ブラウザ=1Bandに簡略化しています。

## セットアップ手順

### 1. GitHubリポジトリを作る

このフォルダ一式(zipを解凍したもの)を、新しいGitHubリポジトリにアップロードしてください。GitHubの「Add file > Upload files」画面はフォルダをドラッグ&ドロップすると構造を保ったままアップロードできます。

`node_modules` と `dist` フォルダはアップロード不要です(Cloudflare Pages側で自動生成されます)。`.gitignore` を同梱していますが、Web UIからアップロードする場合は単純にこの2つのフォルダを含めないようにしてください。

### 2. Cloudflareのアカウント・R2バケットを準備

1. Cloudflareダッシュボードにログイン(アカウントがなければ作成)
2. 左メニューの **R2 Object Storage** から新しいバケットを作成(例: `band-practice-review`)

### 3. Cloudflare Pagesプロジェクトを作成

1. **Workers & Pages** > **Create application** > **Pages** > **Connect to Git**
2. 先ほど作成したGitHubリポジトリを選択
3. ビルド設定:
   - Framework preset: `None` (または `Vite`)
   - Build command: `npm run build`
   - Build output directory: `dist`
4. デプロイを実行(初回は数分かかります)

### 4. R2バケットのバインディングを設定

1. 作成したPagesプロジェクトの **Settings > Bindings > Add > R2 bucket**
2. Variable name: `BUCKET`
3. R2 bucket: 手順2で作ったバケットを選択
4. 保存後、**もう一度デプロイ**し直してください(バインディングは再デプロイしないと反映されません)

### 5. 動作確認

発行されたURL(`https://xxxx.pages.dev`)を開き、Android Chromeと Desktop Chromeの両方で以下を確認:

- 録音ファイルのアップロード→自動分割→保存
- Track再生・コメント追加・★変更
- 「共有」→発行された閲覧用リンクを別ブラウザ(シークレットウィンドウなど)で開いて内容が見えるか
- 編集用リンクでコメント追加・お気に入り変更ができるか
